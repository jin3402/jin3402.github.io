> **"사용자의 여행 성향 분석부터 필수 정보 제공까지, 매끄러운 사용자 경험(UX)에 집중한 크로스 플랫폼 프로젝트입니다."**

## 🙋‍♂️ My Role & Contribution

이 프로젝트에서 저는 **프론트엔드 UI/UX 구현 및 아키텍처 설계**를 담당했습니다. 단순히 화면을 그리는 것에 그치지 않고, 유지보수와 안정성을 고려한 설계에 집중했습니다.

- **Flutter 기반 모바일 앱 개발:** 전체 화면 UI 및 인터랙션 구현
- **위젯(Widget) 모듈화:** 기능 단위 분리를 통한 코드 재사용성 및 가독성 향상
- **방어적 UI(Defensive UI) 설계:** 데이터 로딩 에러 및 화면 깨짐 방지 로직 구축

---

## ✨ UI/UX Implementation (주요 구현 내용)

### 📝 사용자 참여형 인터랙티브 UI (여행 성향 테스트)
- 7단계의 MBTI 스타일 테스트 화면을 상태 관리(State)를 통해 매끄럽게 전환되도록 구현했습니다.
- 테스트 결과에 따른 동적 UI 렌더링으로 맞춤형 여행지를 직관적으로 시각화했습니다.

### 🗺️ 시각적 계층을 고려한 여행지 대시보드
- `ListView`를 활용하여 가로 스크롤 기반의 부드러운 목적지 탐색 경험을 제공합니다.
- `Stack` 위젯을 사용해 이미지 위에 국가별 국기(SVG/PNG)를 오버레이하여 시각적 직관성을 높였습니다.
- 날씨, 시간, 관광지 리스트를 정보 중요도에 따라 컴포넌트화하여 배치했습니다.

### 💱 데이터 가독성을 높인 정보 뷰어
- 다양한 국가의 환율 정보를 사용자가 한눈에 읽기 쉽도록 `Grid/List` 형태로 정렬했습니다.
- 위급 상황을 대비해 `BottomNavigationBar`와 큼직한 버튼 위주의 긴급 연락망 화면을 설계했습니다.

## 🏗 Front-End Highlights (개발 주안점)

### 1. 위젯 모듈화를 통한 유지보수성 극대화
복잡한 화면 구조를 단일 `build` 메서드에 작성하지 않고, `_buildAppBar`, `_buildDestinationCard`, `_buildHorizontalListSection` 등 **논리적인 기능 단위로 분리**했습니다. 이는 코드 가독성을 높이고 협업 효율을 극대화합니다.

### 2. 에러 핸들링을 통한 방어적 UI (Defensive UI)
네트워크 지연이나 이미지 로딩 실패 시 레이아웃이 깨지는 것을 방지하기 위해 `Image.network`의 `errorBuilder`를 활용했습니다. 로딩 실패 시 **플레이스홀더(대체 아이콘)**를 띄워 사용자 경험(UX) 저하를 막았습니다.

### 3. API 연동을 대비한 동적 렌더링 구조
향후 실제 API 연동이 즉시 가능하도록 데이터를 `List<Map<String, String>>` 구조로 캡슐화하여 `ListView.builder`에 연결했습니다. 이를 통해 백엔드와의 협업 시 최소한의 코드 수정으로 데이터를 연동할 수 있는 확장성을 확보했습니다.
