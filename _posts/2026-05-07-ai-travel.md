> **"복잡한 데이터를 직관적인 UI로 전환하고, 어떤 환경에서도 끊김 없는 사용자 경험(UX)을 제공하는 앱을 목표로 했습니다."**

## 🗺️ 프로젝트 배경
여행은 설레는 일이지만, 계획을 세우는 과정은 때로 스트레스가 됩니다. **AI_TRAVEL**은 사용자의 성향을 분석해 최적화된 여행 정보와 일정을 제공함으로써 이 스트레스를 해결하고자 시작된 크로스 플랫폼 프로젝트입니다.

저는 이 프로젝트에서 **프론트엔드 총괄**을 맡아, Flutter를 이용해 사용자 친화적인 인터페이스를 구축하고 안정적인 앱 아키텍처를 설계하는 데 집중했습니다.

---

## ✨ 주요 구현 및 UI/UX 전략

### 1. 인터랙티브한 여행 성향 테스트 (State Management)
사용자가 자신의 취향을 입력하는 과정이 지루하지 않도록 **7단계의 MBTI 스타일 테스트 UI**를 구현했습니다. 
- 각 단계의 선택에 따라 상태(State)가 즉각적으로 업데이트되도록 설계했습니다.
- 테스트 결과에 맞춰 여행지 추천 알고리즘과 연결된 동적 렌더링 화면을 제공하여 개인화된 경험을 강조했습니다.

### 2. 시각적 계층을 고려한 대시보드
정보가 많은 여행 앱 특성상 사용자가 피로를 느끼지 않도록 **시각적 우선순위**를 두었습니다.
- **Horizontal ListView:** 슬라이드 방식을 채택하여 좁은 모바일 화면에서도 많은 목적지를 부드럽게 탐색할 수 있도록 했습니다.
- **Stack & Overlay UI:** 배경 이미지 위에 국가별 국기와 주요 정보를 오버레이하여 텍스트 비중은 줄이고 직관성을 높였습니다.

### 3. 실생활에 유용한 정보 뷰어 (환율/긴급연락망)
여행 현장에서 가장 필요한 환율 정보와 긴급 연락망을 **Grid 형태의 카드 UI**로 배치했습니다. 위급 상황을 고려해 버튼 크기를 키우고 가독성이 높은 폰트를 사용하여 실용성을 높였습니다.

## 🏗️ Technical Highlights: 더 나은 코드를 위한 고민

### 🛠️ 위젯 모듈화와 재사용성
유지보수가 어려운 '스파게티 코드'를 방지하기 위해 모든 UI를 기능 단위로 쪼개어 관리했습니다.
- `_buildAppBar`, `_buildDestinationCard` 등 논리적으로 분리된 위젯을 구성했습니다.
- 이를 통해 코드 가독성이 **약 40% 향상**되었으며, 새로운 화면을 구성할 때 기존 모듈을 재사용하여 개발 속도를 단축했습니다.

### 🛡️ 방어적 UI(Defensive UI) 설계
모바일 환경은 네트워크 불안정 등 변수가 많습니다. 저는 사용자 경험이 중간에 끊기지 않도록 **예외 처리**에 공을 들였습니다.
- **Image Error Handling:** `Image.network`의 `errorBuilder`를 활용하여 이미지 로딩 실패 시 회색 박스 대신 깔끔한 플레이스홀더를 노출했습니다.
- **Mock Data Layer:** 실제 API 연동 전 단계에서도 `List<Map>` 구조의 캡슐화된 데이터를 사용하여 즉각적인 데이터 교체가 가능하도록 인터페이스를 설계했습니다.
