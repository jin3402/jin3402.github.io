## 프로젝트 개요

-팀프로젝트-
여행 계획 단계에서 발생하는 정보 과부하 문제를 해결하기 위해, 사용자의 성향을 분석하고 맞춤형 일정을 제안하는 **AI_TRAVEL** 프로젝트를 진행했습니다. Flutter를 사용하여 모바일 환경에 최적화된 인터페이스를 구축하고 전반적인 앱 구조를 설계하는 역할을 맡았습니다.

---

## 주요 구현 내용

### 1. 성향 테스트 및 개인화 UI
사용자의 취향을 파악하기 위해 7단계로 구성된 MBTI 스타일의 테스트 화면을 제작했습니다.
- **상태 관리:** 사용자가 선택지를 클릭할 때마다 상태(State)가 즉각 업데이트되어 다음 단계로 매끄럽게 넘어가도록 구현했습니다.
- **동적 렌더링:** 테스트 완료 후, 저장된 사용자 성향 데이터에 따라 추천 여행지 화면이 다르게 노출되도록 설계하여 개인화된 경험을 제공했습니다.

### 2. 정보 계층을 고려한 대시보드
모바일의 좁은 화면에서 많은 정보를 효율적으로 전달하기 위해 시각적 계층을 나누는 데 집중했습니다.
- **수평 스크롤(Horizontal ListView):** 여러 목적지를 한눈에 훑어볼 수 있도록 가로 슬라이드 방식을 채택했습니다.
- **스택 오버레이(Stack & Overlay):** 배경 이미지 위에 텍스트와 국기 아이콘 등을 겹쳐 배치하여 정보 밀도를 높이면서도 직관성을 유지했습니다.

### 3. 실용적인 정보 뷰어
여행 현장에서 즉시 확인이 필요한 환율 정보와 긴급 연락망을 카드 UI 형태로 배치했습니다. 위급 상황에서의 가독성을 고려해 버튼 크기를 키우고 명도가 높은 폰트를 사용하여 실용성을 높였습니다.

---

## 개발 시 고민한 점

### 위젯 모듈화와 재사용성
코드의 복잡도를 낮추고 유지보수를 용이하게 하기 위해 모든 UI를 기능 단위로 쪼개어 관리했습니다.
- `AppBar`, `DestinationCard` 등 반복적으로 사용되는 요소들을 별도의 클래스나 함수로 분리했습니다.
- 이를 통해 코드 가독성이 개선되었고, 새로운 화면을 구성할 때 기존 모듈을 재사용하여 개발 효율을 높일 수 있었습니다.

### 방어적 UI(Defensive UI) 설계
모바일 환경의 변수(네트워크 불안정 등)에 대응하기 위해 예외 처리 로직을 강화했습니다.
- **이미지 로딩 예외:** 네트워크 이미지 로딩 실패 시 에러 화면이 뜨는 대신 `errorBuilder`를 활용해 깔끔한 플레이스홀더를 노출하여 UX 저하를 막았습니다.
- **데이터 캡슐화:** 실제 API 연동 전에도 Mock Data를 활용해 앱의 모든 로직이 정상 작동하는지 검증할 수 있도록 데이터 구조를 설계했습니다.
